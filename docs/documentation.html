CSE 3341 Project 2: Full Tokenizer - Ben Powell

User Manual

This project serves as a tokenizer for the Core language, converting a program written in that language to corresponding tokens. The token numbers are sent to the standard output stream for user verification. The Core language cosists of the following legal tokens, all of which are handled and printed in the TokenizerTest class:

  • Reserved words (11 reserved words):
    program, begin, end, int, if, then, else, while, loop, read, write
  • Special symbols (19 special symbols):
    ; , = ! [ ] && || ( ) + - * != == < > <= >=
  • Integers (unsigned, possibly with leading zeros)
  • Identifiers: start with uppercase letter, followed by zero or more uppercase letters and ending with zero or more digits.
For the purposes of this project, these tokens are numbered 1 through 11 for the reserved words, 12 through 30 for the special symbols, 31 for integer, and 32 for identifier. One other useful token is the EOF token (for end-of-file); let us assume that is token number 33.

The code for this project was written in java, and conists of three main files, based on the template provided by Dr Heym. The three neccessary files are :
  • Tokenizer.java - Contains the Tokenizer class which has public methods skipToken and getToken
  • TokenizerTest.java - Contains main method. used to run tests on Tokenizer
  • TokenKind.java - Contains TokenKind inormation used by other classes to differentiate tokens.

Instructions:

  • unzip Interpreter.zip and enter the resulting folder
  • compile: javac -d bin src/edu/c3341/*.java
  • move into bin folder (cd bin)
  • run the command specified in Runfile ( java edu/c3341/TokenizerTest ) with the program text as the command line arg
